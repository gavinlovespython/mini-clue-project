# design notes

this file collects small ideas and observations about the clue system.

## purpose

to store early thoughts that are not ready for spec.md or architecture.md.

## current notes

- the system should stay lightweight and easy to extend
- consider adding optional metadata to clues in the future
- demos should remain isolated and never influence core engine behavior
- configuration should stay external and human-readable
- keep the architecture flexible enough for multiple input sources later
- maintain a consistent tone across all demo languages
